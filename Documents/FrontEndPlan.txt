**خطة إضافة نظام الصفحات والصفحة الرئيسية باستخدام React Router DOM**

---

## **1. هيكل نظام التوجيه والصفحات**

### **هيكل مجلدات الصفحات الجديد**
```
src/
├── pages/                       # مكونات الصفحات الكاملة
│   ├── public/                 # الصفحات العامة (غير مسجّل الدخول)
│   │   ├── Home/              # الصفحة الرئيسية
│   │   ├── About/             # صفحة عن التطبيق
│   │   ├── Contact/           # صفحة الاتصال
│   │   └── Team/              # صفحة الفريق
│   ├── auth/                   # صفحات المصادقة
│   │   ├── Login/             # تسجيل الدخول
│   │   └── Register/          # إنشاء حساب
│   └── protected/              # الصفحات المحمية (بعد التسجيل)
│       ├── Dashboard/         # لوحة التحكم
│       ├── Patients/          # إدارة المرضى
│       ├── MedicalRecords/    # السجلات الطبية
│       └── AIAssistant/       # المساعد الذكي
```

---

## **2. إعدادات React Router DOM**

### **هيكل التوجيه الرئيسي (App Router)**
```javascript
// هيكل المسارات الرئيسية
const router = createBrowserRouter([
  {
    path: "/",
    element: <PublicLayout />,
    children: [
      { index: true, element: <HomePage /> },
      { path: "about", element: <AboutPage /> },
      { path: "contact", element: <ContactPage /> },
      { path: "team", element: <TeamPage /> },
    ]
  },
  {
    path: "/auth",
    element: <AuthLayout />,
    children: [
      { path: "login", element: <LoginPage /> },
      { path: "register", element: <RegisterPage /> },
    ]
  },
  {
    path: "/app",
    element: <ProtectedLayout />,
    children: [
      { index: true, element: <DashboardPage /> },
      { path: "patients", element: <PatientsPage /> },
      { path: "patients/:id", element: <PatientDetailsPage /> },
      { path: "medical-records", element: <MedicalRecordsPage /> },
      { path: "ai-assistant", element: <AIAssistantPage /> },
    ]
  }
]);
```

### **مكونات التخطيط (Layout Components)**

#### **التخطيط العام (PublicLayout)**
- شريط تنقل علوي مع روابط للصفحات العامة
- تذييل (Footer) بالمعلومات الأساسية
- خلفية ونمط موحد للصفحات العامة

#### **تخطيط المصادقة (AuthLayout)**
- تصميم مركزي للنماذج
- روابط للتبديل بين التسجيل والدخول
- خلفية بسيطة ومركزة

#### **التخطيط المحمي (ProtectedLayout)**
- شريط جانبي للتنقل بين أقسام التطبيق
- شريط علوي مع معلومات المستخدم والإشعارات
- منطقة محتوى رئيسية

## **3. الصفحة الرئيسية (Home Page)**

### **مكونات الصفحة الرئيسية**
```
pages/public/Home/
├── components/
│   ├── Hero/                  # قسم البطل الرئيسي
│   ├── Features/              # قسم الميزات
│   ├── HowItWorks/            # قسم آلية العمل
│   ├── Testimonials/          # قسم آراء المستخدمين
│   └── CTASection/            # قسم دعوة للإجراء
├── hooks/
│   └── useHomeAnimations.js   # hooks للحركات والتفعيل
└── HomePage.jsx               # الصفحة الرئيسية
```

### **هيكل الصفحة الرئيسية**

#### **1. قسم البطل (Hero Section)**
- عنوان رئيسي جذاب يوضح فكرة التطبيق
- وصف مختصر وقوي للمشكلة والحل
- زر رئيسي للتسجيل/الدخول
- زر ثانوي لمعرفة المزيد
- صورة أو رسوم توضيحية معبرة
- إحصائيات مثبتة للمصداقية (عدد الأطباء، المرضى، etc.)

#### **2. قسم الميزات (Features Section)**
- شبكة من البطاقات تعرض الميزات الرئيسية
- **بطاقة إدارة المرضى**: تنظيم كامل لبيانات المرضى
- **بطاقة الذكاء الاصطناعي**: تحليل الصور والتشخيص المساعد
- **بطاقة الخطط العلاجية**: وضع ومتابعة الخطط العلاجية
- **بطاقة التقارير**: تقارير شاملة وتحليلات
- كل بطاقة تحتوي على:
  - أيقونة معبرة
  - عنوان واضح
  - وصف مختصر
  - رابط للمزيد

#### **3. قسم آلية العمل (HowItWorks Section)**
- خطوات واضحة ومبسطة لاستخدام النظام
- **الخطوة 1**: التسجيل وإنشاء الحساب
- **الخطوة 2**: إضافة المرضى والبيانات الأساسية
- **الخطوة 3**: رفع الصور والتحاليل
- **الخطوة 4**: الحصول على تحليلات الذكاء الاصطناعي
- **الخطوة 5**: متابعة الخطط العلاجية والتقارير
- تصميم زمني (Timeline) لعرض الخطوات

#### **4. قسم الفريق (Team Section)**
- شبكة من بطاقات أعضاء الفريق
- كل بطاقة تحتوي على:
  - صورة العضو
  - الاسم والمنصب
  - وصف مختصر للتخصص والخبرة
  - روابط التواصل (LinkedIn, Twitter, etc.)
- قسم منفصل للشركاء والداعمين

#### **5. قسم الاتصال (Contact Section)**
- نموذج اتصال متكامل يحتوي على:
  - حقل الاسم
  - البريد الإلكتروني
  - الموضوع
  - الرسالة
- معلومات الاتصال البديلة:
  - البريد الإلكتروني الرسمي
  - رقم الهاتف
  - العنوان
- خريطة للموقع (إن وجد)

#### **6. قسم دعوة للإجراء (Call to Action)**
- رسالة تشجيعية قوية
- زر رئيسي للتسجيل الآن
- ضمان تجريبي أو عرض ترويجي
- أسئلة شائعة مصغرة

---

## **4. نظام التنقل بين الصفحات**

### **شريط التنقل الرئيسي (Navbar)**
- شعار التطبيق (يؤدي للصفحة الرئيسية)
- روابط الصفحات العامة (عن التطبيق، الفريق، الاتصال)
- أزرار الدخول والتسجيل
- في حالة المستخدم المسجّل:
  - زر للدخول للوحة التحكم
  - قائمة المستخدم مع خيارات
  - إشعارات

### **شريط التنقل الجانبي (Sidebar)**
- مجموعة روابط الأقسام الرئيسية
- أيقونات معبرة لكل قسم
- حالات نشطة للرابط الحالي
- مجموعة مختصرة للمستخدم

### **تذييل الصفحة (Footer)**
- روابط سريعة للصفحات المهمة
- معلومات الاتصال
- روائل التواصل الاجتماعي
- الشعار وحقوق النشر

---

## **5. تحسينات تجربة المستخدم**

### **حركات وتفاعلات (Animations)**
- حركات ظهور متدرجة للمحتوى
- تأثيرات hover على البطاقات والأزرار
- حركات انتقال سلسة بين الصفحات
- مؤشرات تحميل أثناء الانتقال

### **التصميم المتجاوب (Responsive Design)**
- تكيف مع جميع أحجام الشاشات
- قائمة هامبرجر للشاشات الصغيرة
- إعادة ترتيب المحتوى حسب أولوية العرض
- تحسين تجربة اللمس للهواتف

### **تحسين سرعة الصفحة**
- تحميل بطيء للصور (Lazy Loading)
- تحسين حجم وتحميل الخطوط
- تقليل حجم الحزم (Bundle Size)
- استخدام CDN للصور والموارد

## **7. خطة تنفيذ الصفحات**

### **المرحلة 1: الصفحات العامة (أسبوعان)**
- الصفحة الرئيسية (Home) مع جميع الأقسام
- صفحة عن التطبيق (About)
- صفحة الفريق (Team)
- صفحة الاتصال (Contact)
- نظام التنقل الرئيسي والتذييل

### **المرحلة 2: صفحات المصادقة (أسبوع)**
- صفحة تسجيل الدخول
- صفحة إنشاء الحساب
- تخطيط المصادقة الموحد

### **المرحلة 3: الصفحات المحمية (3 أسابيع)**
- لوحة التحكم الرئيسية
- صفحة إدارة المرضى
- صفحة السجلات الطبية
- صفحة المساعد الذكي
- التخطيط المحمي والشريط الجانبي

---

## **8. تكامل النظام**

### **التوجيه الشرطي (Conditional Routing)**
- إعادة توجيه المستخدم المسجّل من الصفحات العامة للوحة التحكم
- إعادة توجيه المستخدم غير المسجّل من الصفحات المحمية لتسجيل الدخول
- حفظ المسار المحاول للعودة إليه بعد التسجيل

### **حماية المسارات (Route Protection)**
- مكون ProtectedRoute للتحقق من التوثيق
- مكون PublicRoute لمنع الوصول للصفحات العامة بعد التسجيل
- تحقق من الصلاحيات للمسارات الحساسة

### **تحسين محركات البحث (SEO)**
- عناوين وصفوف فريدة لكل صفحة
- بيانات structured data للمحتوى
- روابط canonical
- sitemap.xml
